1. Разработать классы для управления магазином, классы следует связать между собой:

Обязательные классы:
    класс Product, обязательные элементы — имя, себестоимость, цена и вес товара (базовый класс);
    класс Customer, обязательные элементы — имя, скидка и валюта покупателя(выбор из нескольких вариантов в системе);
    класс  Buy, содержит данные о количестве товара, о цене за единицу, о цене за весь купленный товар и  о весе товара (производный класс для класса Product и базовый класс для класса Check);
    класс Check, не содержащий никаких полей. Данный класс должен выводить на экран информацию о товаре и о покупке, с учетом валюты покупателя и компании (производный класс для класса Buy);
    
    Приложение консольное, меню должно давать возможность выбора действий: 
    1) регистрация  продуктов, покупателей
    2) продажа продуктов
    3) подведение итога
